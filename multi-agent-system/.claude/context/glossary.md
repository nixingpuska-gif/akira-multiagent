# Glossary - Глоссарий Терминов

## Термины Системы Агентов

**Agent** - автономный AI-агент с определенной ролью и ответственностью

**HR Manager** - агент, отвечающий за создание и настройку других агентов

**Product Manager** - главный агент-координатор, единственная точка контакта с пользователем

**Researcher** - агент, ответственный за поиск информации и исследования

**Architect** - агент, ответственный за архитектурные решения

**UX Designer** - агент, ответственный за UX/UI дизайн

**Visual Designer** - агент, ответственный за визуальный дизайн

**Frontend Developer** - агент (Codex), пишущий frontend код

**Backend Developer** - агент (Codex), пишущий backend код определенного модуля

**Tester** - агент, проверяющий соответствие acceptance criteria

**Validator** - агент, единственный кто может утвердить выполнение задачи

---

## Термины Процесса

**Task** - задача с четко определенными acceptance criteria

**Acceptance Criteria** - измеримые критерии, по которым оценивается выполнение задачи

**Proof of Work** - доказательства выполнения работы (файлы, тесты, метрики, hash)

**Hash Chain** - цепочка хэшей для audit trail и прослеживаемости

**ADR (Architecture Decision Record)** - документ, описывающий архитектурное решение

**File Lock** - блокировка файла для предотвращения конфликтов

**Audit Trail** - полный журнал всех действий агентов

---

## Термины Дизайна

**Wireframe** - низкодетализированный макет интерфейса

**Mockup** - высокодетализированный визуальный макет

**Design System** - набор переиспользуемых компонентов и стилей

**UI Kit** - библиотека UI компонентов

**Moodboard** - коллекция референсов и вдохновения для дизайна

**Glassmorphism** - дизайн-стиль с эффектом "стекла"

---

## Термины Разработки

**Module** - независимая часть системы с четкими границами

**Code Style** - набор правил оформления кода

**Test Coverage** - процент кода, покрытого тестами

**Refactoring** - изменение структуры кода без изменения функциональности

---

## Статусы Задач

**inbox** - новая задача, еще не назначена

**in-progress** - задача в работе

**review** - задача завершена, ожидает проверки

**completed** - задача утверждена Validator

**blocked** - задача заблокирована

**rejected** - задача отклонена, требует доработки
